关于python中的re.sub()函数

发表于 2016-06-14   |   分类于 python

sub(pattern, repl, string, count=0, flags=0)
　　Return the string obtained by replacing the leftmost
　　non-overlapping occurrences of the pattern in string by the
　　replacement repl. repl can be either a string or a callable;
　　if a string, backslash escapes in it are processed. If it is
　　a callable, it’s passed the match object and must return
　　a replacement string to be used.

个纯文本文件添加一些格式，此章会给文本添加HTML标记。

yield的作用就是将一个函数变成一个生成器（生成器相当于自定义迭代规则的迭代器），
函数执行到yield时，便会返回一个迭代值，下次迭代时，代码从下一句开始执行，而函数的
本地变量看起来和上次中断执行前是完全一样的，于是函数继续执行，直到再次遇到yield。
我个人理解为，遇到yield时将会停在此处，交给上一级处理，处理完毕后回到此处继续向下
运行。

handlers.py

这是一个处理程序，用来负责产生标记文本，但也接受来自文本分析器的具体指令。
Hanlder类隐藏一些具体管理细节，不需要块的全名进行方法调用，而是将块类型当
作字符串处理并将其提供给处理程序，并可以用callback方法检测对应方法是否真的
被实现了，如果没有找到对应方法，便什么也不做（不输出开始结束标记和原样输出
文本块）。

  • start，end方法
    对每个类型的块都有一对方法，一个用于开始块，一个用于结束块（进行开始结束
    标记）。调用start，end方法，只要识别出这段文本块的类别，就能通过类别名字添
    加对应开始结束标记。
  • sub方法
    能识别出一段文本块中需要在html特殊处理的文字（如url链接，邮箱链接，网站链
    接），然后通过正则表达式的方法替换成html中的对应标记文本。
  • feed方法
    用于将分析过后的实际文本交给处理程序去处理，简单的html渲染程序中即为直接输
    出。

rules.py

规则是用来进行语法分析的，在主程序中使用，主程序要决定对给定的块使用什么样的规则，让
每个规则对块做需要的转换，即规则需具备如下功能：

  • 能识别自己适用于哪种块（condition）
  • 能对块进行转换（action）

conditon，action方法都需要块block作为参数，condition需要返回一个布尔值来表示规则是否
适用于给定的块，而action方法需要返回一个布尔值来表示是否应该停止对当前块进行规则处理。

markup.py

主程序中也包含着本程序核心-Parser类，它使用一个处理程序，一系列规则和过滤器将纯文本文件
转换成标记文件（本程序中是HTML文件），则Parser类需要一个负责初始化的构造器，一个添加规则
的方法，一个添加过滤器的方法，以及一个对给定文件进行语法分析的方法。

  • 构造函数只是将提供的处理程序分配给一个变量，然后初始化两个列表（一个规则列表，一个过滤
    器列表）。

    但addFilter在向过滤器列表添加一个过滤器之前先要创建那个过滤器。过滤器只是一个函数，此函
    对合适的正则表达式（PATTERN）用re.sub进行替换，替换的内容repl来自处理程序，处理程序使用
    hander.sub(name)进行访问。这也是我产生理解问题的部分
  • Parser方法从调用start(‘document’)开始，到调用end(‘document’)结束，中间迭代文本文件中
    的所有块。对每个块使用过滤器和规则。使用过滤器只是用块和处理程序做参数来调用filter函数，
    并且会重新把块绑定到结果。如：block=filter(block,self.handler)

**

对于markup.py中15行的代码，起初我认为是层层调用并return，最终这句话可写成
re.sub(r’\(.+?)\‘,’%s<\em>’%match.group(1),block)，因而不太能理解match的定义赋值方
式，后来查询了re.sub()的参数解释，误以为此处函数会将匹配对象传递给match，因而这句话得以
运行。但最后却发现此处’%s<\em>’%match.group(1)为一个字符串，因为并不会被传递匹配对象。
比如，在python解释器中运行re.sub(r’*(.+?)*‘,’%s<\em>’%match.group(1),block)，仍然得
到报错：match未定义。
后经询问后发现，此处15行代码不应return代入后理解，而是：handler.sub(name)会返回substitution
的函数名，此处为re.sub(r’*(.+?)*‘,substitution,block)，中间为一函数，因而此行代码运行时
匹配对象会传递给substitution函数，作为match的值，从而成功运行。
比如，在python解释器中运行re.sub(r’*(.+?)*‘,sub_emphasis,’abc123abc’)（前面已输入
sub_emphasis(match)的函数），会得到abc123abc。
